Inline Styles#

Inline styles set via the HTML ‚Äãstyle attribute are processed by the ‚Äã@native-html/css-processor library. You don't need to wonder if a CSS property will break your app: the CSS processor acts as a compatibility layer between React Native styles and CSS properties. This library gives you leverage on inline CSS processing via multiple props:

‚ÄãenableCSSInlineProcessing

Disable inline styles processing altogether.

‚ÄãallowedStyles

Whitelist the camel-cased CSS properties that you wish to be included.

‚ÄãignoredStyles

Blacklist the camel-cased CSS properties that you wish to be excluded.

Props#

The ‚ÄãRenderHTML component has four props to customize elements styles:

‚ÄãbaseStyle

The styles for the root component. Inheritable styles will be inherited by all children.

‚ÄãidsStyles

Target elements with the HTML ‚Äãid attribute.

‚ÄãclassesStyles

Target elements with the HTML ‚Äãclass attribute;

‚ÄãtagsStyles

Target elements by tag name.

Each of these props is a record mapping identifiers with a ‚ÄãMixedStyleDeclaration.

note

There is not (yet) ways to provide CSS selectors to target elements. Because it would require a full-featured CSS parser to build an AST, this feature might be supported in the future with an extension.

Mixed Style Records#

Introduction#

A mixed style declaration is an object similar to ‚ÄãStyleSheet's create method argument values. However, it supports a blend of React Native ViewStyle, TextStyle and camel-cased CSS properties. See ‚ÄãCSS Processing for a complete reference.

A mixed styles declaration record.
const tagsStyles = {
body: {
whiteSpace: 'normal',
color: 'gray'
},
a: {
color: 'green'
}
}

In the above snippet, mixed-style declarations are the objects defined as values of the tagsStyles record. A few important comments:

  1. Line 2. The ‚Äã<body> will always be present and can be confidently targeted, event when missing in the HTML snippet.

  2. Line 3. This is the default property, with the exception of ‚Äã<pre> and a few other tags.

  3. Line 4. Thanks to CSS inheritence, all textual children of ‚Äã<body> will default to this property.

  4. Line 7. The ‚ÄãTRE implements CSS specificity. A rule targetting the ‚Äã<a> element will override rules defined for its ancestors (in this case, ‚Äã<body>).

warning

You should never provide styles from React Native ‚ÄãStyleSheet. These will not work.
